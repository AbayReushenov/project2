Why do databases need to adapt?

91 comments

As organisations grow, it may be necessary for a database to change, for example to:

  • Add more data about computer models
  • Store and organise customer contact details

Altering a table

In the sales table of your computer sales database, there isn’t a field to store how much a computer was sold for.

By adding a price field to the table you could keep track of the income of the company and search for the most profitable models.

You can use SQL to modify a table’s structure and add a new field.

  1. Execute the SQL statement below:

    ALTER TABLE sales
ADD price real;

    The ALTER TABLE statement is used to update an existing table. The name of the table entered after the ALTER TABLE statement must exist or an error will occur.

    This statement is followed by the changes to be made. In the example above, you added price as a new field in the model table, with the data type real.

  2. If you view the data in the sales table you will see that there is no data in the price field.

You need to use UPDATE, SET, and WHERE clauses in an SQL statement to update the model table by setting the price field to the cost of the computer.

UPDATE sales
SET price = 35
WHERE sale_id = 1;

Why add price to sales?

In this example, I chose to add the price field to the sales table. An alternative would have been to add the price field to the model table. Doing so would have resulted in less data being stored, as the price would only have been stored once for each model, rather than once per sale.

Why did I choose to put it in the sales table and not the model table? Discuss your thoughts in the comments section.
